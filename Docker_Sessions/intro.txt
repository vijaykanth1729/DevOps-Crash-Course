----------
python 3.10

Python 3.8

Result: Application Crashes

Docker:
----------
Docker packages:
 Application
 Dependecnies
 Libraries
 Runtime

Into a single unit which we call as Container.

VM = Full house, Container - = Flat in apartment

Docker Architecture:
----------------------
1) Docker client: (CLI) to run commands in that cli.

2) Docker Daemon
    - Background service this does the actual work.

3) Docker Images
   - BluePrint / Template.

4) Docker container
   - Running instance of an image.

5) Docker Registry
   - store images 
   ex: Docker Hub
    AWS ECR.

Flow Diagram:
-------------
User-->Docker cli-->Docker Daemon -->Pull Image-->Run Container

Docker Setup:
-------------
Follow Docker URL : https://docs.docker.com/desktop/setup/install/windows-install/

Run First Container:
--------------------

docker run hello-world (Image called hello-world) {Run command internally pulls the image and run container out of it}
docker pull hello-world
docker images
docker rmi image_name (Removes the image from local state)
docker ps (shows which containers are running)
docker ps -a  (shows containers which are running and stopped)


To try something more ambitious, you can run an Ubuntu container with:
 $ docker run -it ubuntu bash
